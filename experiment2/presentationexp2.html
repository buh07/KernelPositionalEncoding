How a Hypothesis Gets Marked Pass vs Fail

Step-by-step adjudication

  1. Effect must be large enough: accuracy drop ≥ 0.05 or relative NLL increase ≥ 0.10.
  2. Directional contrast must hold: class difference ≥ 0.05 in the predicted direction.
  3. Targeted ablation must beat random control by at least 0.02.
  4. Kernel co-occurrence threshold must be met: |ΔR²| ≥ 0.03 on Track A or Track B raw.
  5. Uncertainty must support the claim: BCa CI and Holm-adjusted sign-flip tests.

Precision labels

  • pass: threshold met, CI lower bound > 0.
  • imprecise_pass: threshold met, CI overlaps 0.
  • fail: threshold not met or direction reversed.

Selectivity labels

  • strong_directional_support
  • directional_nonselective
  • no_directional_support

These distinguish "right direction but broad degradation" from truly selective effects.

Boundary: H3 is reported as co-occurrence/specificity, not mediation identification.

Intervention Mechanism (Deterministic and Auditable)

How channels are selected

  • D = d_head / 2 rotary channel pairs.
  • High band = top half by rotation rate.
  • Low band = bottom half by rotation rate.
  • Medium/strong severities remove 40%/60% of the targeted band.
  • Nestedness: medium mask is always a subset of strong mask.

Random controls

  • Count-matched random masks for medium and strong severities.
  • Three random draws per seed in confirmatory phases.

Audit rules

  • Norm-preserving Q/K rescale after masking.
  • Median norm-ratio drift tolerance: ≤ 0.05.
  • Failing runs are excluded from confirmatory adjudication.

Specificity statistics: S_ratio = |ΔR2_targeted| / (|ΔR2_random| + 0.01), S_diff = |ΔR2_targeted| - |ΔR2_random|.

Simple reading: We test whether "smart" frequency-targeted damage does more than equally sized random damage.

Task Generators: Detailed but Reproducible

Synthetic tasks (causal core)

  • local_copy_offset (short): copy token from a small backward offset.
  • local_key_match (short): resolve nearest valid key in a local window.
  • delayed_copy (long): copy token from a large offset.
  • long_range_retrieval (long): retrieve mapped value for a distant key query.

Reproducibility controls

  • Deterministic token pools and seed control.
  • Deterministic resampling for invalid or ambiguous examples.
  • Fixed sample counts: synthetic 200, Tier-1 100.
  • Sequence IDs reused within seed across interventions for paired contrasts.

Tier-1 transfer setup (exploratory)

  • Datasets: one wiki-like and one code-like corpus.
  • Dependency typing from local concentration score.
  • 2x4 stratification: dependency type x baseline NLL quartile.
  • Binning validity threshold: separation gap ≥ 0.15.

Simple reading: Tier-1 asks whether the synthetic directional story appears in natural text, but this remains exploratory and interpretability-gated.

Exactly what happens in order

  1. Run baseline first and apply floor/feasibility checks.
  2. If baseline is too weak, apply deterministic fallback profile and re-check.
  3. Run targeted and random conditions with matched sequence IDs per seed.
  4. Write task metrics, kernel summaries, and intervention audits per cell.
  5. Compute pilot directional signal and variance used for Phase 2B MDE context.

What success looks like in Phase 2A

  • Strong targeted effect clears size threshold (acc drop >= 0.05 or rel-NLL >= 0.10).
  • Directional class contrast appears (>= 0.05).
  • Targeted contrast exceeds matched random by >= 0.02.
  • Kernel shift threshold met (|ΔR²| >= 0.03).

What Phase 2A cannot do alone

  • Cannot prove cross-model robustness.
  • Cannot finalize publication-level confirmatory support by itself.

Simple reading: Phase 2A answers one question: “Do we have a real, measurable signal worth replicating?”

Phase 2A outputs: per-cell run_config.json, task_metrics.parquet, kernel summaries, and audit files that feed Phase 2B gating.

Phase 2B decision sequence (plain language)

  1. Require directional replication in at least 2 of 3 RoPE models.
  2. Require within-class sign agreement (both short tasks align; both long tasks align).
  3. Require content-gated contrast threshold (>= 0.03).
  4. Require targeted-vs-random specificity advantage (>= 0.02).
  5. Require H3 pass-rate threshold on confirmatory groups.

Exact gate meaning

  • If criteria 1-5 pass: confirmatory claims are eligible (with full inferential reporting).
  • If any of criteria 1-5 fail: confirmatory claims stop; continuation is exploratory-only.

What this protects against

  • Single-model flukes being presented as robust effects.
  • Copy-only task behavior being over-generalized to all dependency types.
  • Generic perturbation damage being mistaken for frequency specificity.

Simple reading: Phase 2B is the final exam. If replication + specificity fail here, confirmatory claims are blocked.

Phase 2B outputs: pooled inferential summaries for H1/H2, H3 specificity tables, gate decisions, and promotion-eligibility metadata.

Statistical Machinery and Decision Outputs

Inference stack

  • Paired within-seed contrasts (same seed, same cell family).
  • 95% BCa bootstrap CIs (B=20,000).
  • Two-sided paired sign-flip p-values.
  • Holm-Bonferroni correction for the H1/H2 family.

Main outputs

  • gate_evaluation.json
  • decision_summary.json
  • specificity_metrics.parquet

Power and detectability

  • Phase 2A variance feeds Phase 2B MDE summaries.
  • Families marked underpowered if detectable effect is above target threshold.
  • This downgrades claim strength, not thresholds.

Simple reading: effect size + uncertainty + multiplicity correction all have to agree before a strong confirmatory claim is made.
